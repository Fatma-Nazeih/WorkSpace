HTB — Forest

Windows / Active Directory box — anonymous LDAP bind to Domain Admin via a classic AS-REP roasting + group-membership abuse chain.

windowsactive directoryhtb

Recon

nmap -sC -sV -p- 10.10.10.161

Standard DC fingerprint: 53/DNS, 88/Kerberos, 389/LDAP, 445/SMB — confirms this is a domain controller for htb.local.

Foothold — AS-REP Roasting

LDAP allows anonymous bind, which is enough to enumerate usernames. One account, svc-alfresco, has Kerberos pre-authentication disabled — meaning its AS-REP hash can be requested with no credentials at all.

impacket-GetNPUsers htb.local/ -usersfile users.txt -no-pass -dc-ip 10.10.10.161
hashcat -m 18200 asrep.hash rockyou.txt

The cracked password gives a shell as svc-alfresco.

Privesc — group membership abuse

Enumerating group membership with BloodHound shows svc-alfresco is a member of Service Accounts → Privileged IT Accounts, which has GenericAll over the Exchange Windows Permissions group. That group in turn has WriteDacl on the domain object itself — enough to grant DCSync rights to a controlled account.

net rpc group addmem "Exchange Windows Permissions" svc-alfresco -U htb.local/svc-alfresco
# grant DCSync rights via a DACL ACE, then:
impacket-secretsdump htb.local/svc-alfresco:'password'@10.10.10.161

DCSync dumps every domain hash including Administrator — full domain compromise from there.

Takeaways

  • Anonymous LDAP bind is a small misconfig with outsized impact — it's the entire opening move here.
  • Nested group membership is where most real AD privilege chains hide; BloodHound's shortest-path graph found this in seconds versus manually walking ACLs.
