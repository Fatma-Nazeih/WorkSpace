Recon Toolkit

The tools that actually make it into my recon phase, roughly in the order I reach for them.

recontooling

Subdomain enumeration

  • subfinder — fast passive enumeration across many sources, good default first pass.
  • amass — slower, more thorough; worth the extra time on a wide scope.
  • httpx — probe the resulting list for live hosts, status codes, and titles in one pass.

Content discovery

  • ffuf — directory/parameter fuzzing, fast enough to iterate wordlists quickly.
  • gau / waybackurls — pull historical URLs, catches endpoints that no longer link from anywhere but still resolve.
  • SecLists — the wordlist source for basically everything above.

Manual testing

  • Burp Suite — Autorize, Turbo Intruder, and Logger++ are the extensions that stay installed on every engagement.
  • Caido — lighter alternative when Burp's overhead isn't needed.

Active Directory

  • BloodHound — attack path graphing, essential on any internal AD engagement.
  • Impacket suite — secretsdump, GetNPUsers, psexec cover most of the post-foothold workflow.
  • NetExec (formerly CrackMapExec) — spraying, enumeration, and execution across a subnet from one tool.

Notes-to-self

Keep tool output in per-engagement folders named by date + scope, not by tool — makes it far easier to reconstruct a timeline later when writing the report.
